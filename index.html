◆ DHT Globe — User Guide

DHT Globe is a browser-based simulator for Distributed Hash Table protocols. Nodes are placed on a 3-D globe; lookups route messages through the network hop by hop. Classical protocols (Kademlia, Geographic) use static routing tables. Neuromorphic protocols learn from traffic — synaptic shortcuts form on heavily-used paths and decay on idle ones, letting the routing table adapt over time.

Quick Start
  1. Choose a Protocol and set the Node count, then click Init to build the network.
  2. Click Lookup Test to run random key lookups and see hop-count and latency stats.
  3. Click Demo to animate a single lookup — arcs trace each hop on the globe.
  4. Click Train Network to run repeated sessions on any protocol. Classical protocols (Kademlia, Geographic) refine their k-buckets with fresher peer contacts; neuromorphic protocols additionally build synaptic shortcuts and show larger hop-count reductions over time.
  5. Click Benchmark to compare selected protocols side-by-side across selected test conditions.

Drag the globe to rotate · scroll to zoom · click any node to show its routing table connections.

Tests & Modes
ButtonWhat it does
■ InitPlace nodes on land, assign IDs, and populate routing tables. Re-clicking resets everything.
▶ Lookup TestRun Count random key lookups. Reports success rate, hop count (avg / p50 / p95 / max), and latency histograms.
↻ Churn TestSimulate node turnover over multiple intervals. Measures how routing health (success rate, hop count) degrades and recovers as nodes leave and join.
▶ DemoAnimate one lookup on the globe. The camera follows each hop; arcs shift yellow → orange → deep orange with distance.
⟳ Train NetworkRepeat lookup sessions in a loop for any protocol. All protocols improve with training: Kademlia and Geographic DHT update their k-buckets with fresher, more recently-seen peers as traffic flows through the network. Neuromorphic protocols additionally run a full learning pass each session (Markov annealing, hop-caching, triadic closure, LTP), producing larger and more sustained hop-count reductions. Press again to stop. Exports a CSV of all sessions.
⊕ Pub/SubMulti-group pub/sub overlay test. Nodes form concordance groups (one relay + participants, like a Croquet session). A random participant sends a message to its relay; the relay broadcasts back to every participant. Groups overlap — nodes can belong to multiple groups as relay, participant, or both — creating realistic routing-path interference. Runs continuously, recording hops and simulated latency (20 ms/hop) per message cycle. Press again to stop.
⇔ Pair LearningEach node is assigned a fixed random target. Every session routes all pairs and records average hops. Measures how quickly the network memorises persistent routes.
🌡 HotspotTwo-phase load test. Highway: tracks which nodes relay the most traffic (Gini / Lorenz curve). Storage: queries content items with a Zipf distribution and measures query-load concentration.
▶▶ BenchmarkRuns every protocol (or the Fast subset) across many conditions: regional radii, source/dest-restricted, global, continent-crossing, and churn. No Init required.
Control Groups
GroupParameters
NetworkProtocol — routing algorithm. Nodes — network size (20 – 100K). K — bucket size (Kademlia default 20). α — parallel queries per round. Bits — ID bit-width (64 default). Delay — per-node processing latency added to every hop.
LookupCount — lookups per test run. Hot% — traffic concentration: 100% = uniform random; lower values focus traffic on a popular sub-network so neuromorphic protocols learn denser shortcuts.
ChurnRate — % of nodes replaced per interval. Int — number of churn cycles. L/Int — lookups measured per cycle.
FiltersRegion — restrict lookups to targets within a km radius. Src — only the top X% of nodes initiate lookups. Dest — only the top X% of nodes are valid targets.
Pub/SubGroup — participants per concordance group (relay + this many nodes, default 32). Cover % — percentage of network nodes that join at least one group (default 10%). More coverage creates more groups with overlapping routing paths.
HotspotQueries — number of lookups in the highway phase. Items — content items in the storage phase. Zipf — exponent controlling skew (0 = uniform, 1 = classic internet, 2 = extreme).
BenchWarmup — training sessions given to neuromorphic protocols before scoring (each session = 500 lookups). Protocols — multi-select of which protocols to benchmark. Tests — multi-select of which test conditions to run. Both support Select All / Deselect All. Settings are saved between sessions.
Protocols
ProtocolKey idea
KademliaClassical DHT baseline. XOR distance metric; O(log N) hops. K-bucket routing table, α parallel queries per round.
G-DHTAssigns IDs from physical location using an S2 Hilbert-curve prefix. Nearby nodes share high-order bits — routing paths naturally follow geographic clusters. Prefix width configurable via the G-DHT Bits parameter.
N-1First neuromorphic variant. Adds a synaptome (learned routing table) on top of Kademlia. Every hop caches the path — heavily-used routes get stronger synapses; idle ones decay.
N-2 / N-2-BP / N-2-SHCAdds learning algorithms: backpropagation credit assignment and stochastic hill-climbing to guide which synapses reinforce.
N-3Triadic closure — if A routes through B to reach C, A tentatively wires a direct synapse to C, shortcutting B over time.
N-4Markov annealing — periodically scans for candidate peers and promotes or demotes synapses based on observed routing improvement.
N-5Stratified routing table — maintains at most k synapses per distance stratum, ensuring broad coverage at all scales.
N-5WAdds weighted synapse reinforcement calibrated to browser-style traffic patterns (bursty, with hot nodes).
N-6WAdaptive load shedding via exponential moving-average (EMA) backpressure — overloaded nodes signal congestion; traffic redistributes around hotspots.
N-7WHighway formation — high-traffic nodes evolve into dedicated relays. Gini-aware load balancing prevents relay monopolies.
N-8WMulti-path cascade spread — lookups fan out across parallel synaptic paths; LTP (long-term potentiation) backpropagates credit to earlier hops that contributed to successful routes.
N-9WSynaptome floor protection — prevents the routing table from collapsing in random global traffic. Guard A stops annealing from pruning below the floor; Guard B resets decaying entries rather than deleting them when the tier is at or below the floor.
N-10WRelay pinning — frequently-targeted destinations (pub/sub relays, hot topics) are promoted to a protected set immune to eviction and decay. Load-aware routing removed (it raised per-hop latency in relay scenarios). Raised Markov seed weight restores N-6W's fast participant-entry stabilisation for near-perfect broadcast.
N-11WFrequency-weighted synaptome reservation — replaces N-10W's hard 4-pin cap with a dynamic budget scaling to 50% of the local tier (24 slots). Pins are frequency-ranked: when the budget is full, a new hot destination displaces the lowest-frequency current pin. Synaptome entries that qualify by frequency are also treated as eviction-immune before formal registration, closing the coverage gap at high pub/sub load and large network scale.
N-12WHighway-tier relay pinning — moves relay pins from the local synaptome into the highway tier. The local 48-slot routing table is fully restored for XOR diversity; up to 4 of the 12 highway slots are reserved for pinned relay destinations and the remaining 8 for hub nodes. Load-aware AP scoring is also restored from N-9W (removed in N-10W), which reduces relay hop count at large scale by distributing traffic away from saturated relay nodes.
N-13WRelay pinning + load awareness — combines N-10W's local-tier eviction-immune relay pinning (the correct placement: 4 pins = 8.3% of 48 local slots) with N-9W's load-aware AP scoring. N-12W showed that highway-tier pins are counterproductive: displacing 33% of the 12-slot highway with geographically clustered pub/sub members removes long-range routing shortcuts and raises relay hops. N-13W reverts to N-10W's local-tier pinning and simply restores the load discount (LOAD_PENALTY=0.40) that N-10W dropped, targeting the best of both: ~3-hop relay routing from load balancing and ~1.7 bcast hops from eviction-immune relay pins.

Neuromorphic protocols are cumulative — each generation adds one mechanism on top of the previous one. N-13W contains all mechanisms from N-1 through N-9W plus local-tier relay pinning and load-aware routing.
